Conceptos básicos en la terminal

Navegar dentro de la terminal:

Estos comandos sirven para moverse entre las carpetas del computador a través de la terminal:
* pwd: indica la carpeta actual.
* ls: mostrar los archivos dentro de la carpeta actual.
* ls -al: mostrar todos los archivos (incluidos los ocultos) en forma de lista.
* cd +nombre de la carpeta: entrar a una carpeta dentro de la carpeta actual.
* cd .. : ir a la carpeta anterior.
* cd /: ir a la raíz del sistema.
* history: muestra el historial de comandos ingresados.
* flecha hacia arriba: recorre los comandos anteriormente ingresados.
* !# : repite el comando número # ingresado.
* rm "nombre del archivo": elimina por completo el archivo (hay que tener cuidado con este comando).
* alias nombre="comando": se crea un alias para un comando muy extenso.












git:

Este comando sirve para verificar que se haya instalado Git en el PC.




git init:

Este comando sirve para inicializar Git dentro de la carpeta del proyecto a trabajar.




git config —global user.name “usuario(javier)”:

Este comando sirve para establecer el nombre del usuario que está controlando Git.




git config —global user.email “sucorreoregistradoenGitHub@correo.com”:

Este comando sirve para establecer el email del usuario que está controlando Git. ( es absolutamente necesario hacer esto).




git status:

Este comando sirve para verificar el estado de los archivos del proyecto, si fueron modificados, si estan añadidos, o si estan listos para guardar.




git log:

Este comando sirve para ver todo el historial de versiones.
Usar git log --all --graph --decorate --oneline, para ver mejor la historia en la terminal.
Se puede abrir una interfaz gráfica donde se observa mejor la historia usando gitk




git add:

Este comando sirve para añadir archivos al estado de staging.




git commit -am "Mensaje de indicación del commit":

Este comando sirve para pasar los archivos del estado de staging (Temporal) al repositorio (local), quedan fijados como una versión. En otras palabras, es guardar la versión de los cambios realizados.




git reset HEAD:

Nos ayuda a sacar archivos del estado Staged para devolverlos a su estado anterior. Si los archivos venían de Unstaged, vuelven allí.




git reset --hard:

Volver a una versión anterior y borrar las versiones posteriores a esa.




git reset --soft:

Volver a una versión anterior pero mantener vivos los archivos en staging.




git rm --cached:

Mueve los archivos que le indiquemos al estado Untracked.




git rm --force:

Elimina los archivos de Git y del disco duro. Git guarda el registro de la existencia de los archivos, por lo que podremos recuperarlos si es necesario (pero debemos usar comandos más avanzados).




git diff "asqhrjejf" "adbekfjw":

Compara las versiones y muestra las diferencias de los archivos de la primera version con respecto a la segunda nombrada.




git show "archivo":

Muestra los cambios de ese archivo.




git show-ref --tags:

Muestra los tags y su relación de referencia.




git tag:

Muestra los tags existentes.




git tag -a v0.1 -m "Mensaje del tag //es como el mensaje de un commit" + codigo de referencia de git:

Crea un tag llamado v0.1 en el commit correspondiente al codigo de referencia.




git stash

Guardar cambios en memoria y recuperarlos después, esto permite moverse entre ramas mientras está el stash en ejecución sin tener que hacer commit. Al guardar los cambios en memoria, regresa a la versión sin cambios. * list: Lista los cambios que están guardados en memoria o en "stash".
* pop: devuelve al estado sin stash.
* branch +nombre: guarda los cambios en memoria, en una rama (+nombre). Se comitean los cambios en la rama,
* drop: se usa cuando se hicieron cambios por error y se vuelve a la versión sin esos errores con stash, la infomación que esta en memoria (stash), se elimina con git stash drop.

Amend (remendar):

Enmendar errores. Por ejemplo cambios que faltaron hacer en un commit. Para esto
* Se agregan los cambios "git commi add archivo_con_cambios.txt"
* O simplemente git add --all, para agregar todos los cambios.
* Luego se ejecuta en consola: git commit --ammend
* Se confirma el mensaje del commit en vim. Y listo.
