Comandos para uso de GitHub

git remote add origin "url":

Con este comando agregamos la URL de GitHub donde se va a almacenar el repositorio remoto.




git remote -v:

Con este comando verificamos que la URL haya quedado asignada correctamente en el repositorio local.

git pull origin master:

Mostrará este resultado en consola:
warning: no common commits
remote: Enumerating objects: 3, done.
remote: Counting objects: 100% (3/3), done.
remote: Total 3 (delta 0), reused 0 (delta 0), pack-reused 0
Unpacking objects: 100% (3/3), done.
From https://github.com/dariofgomez/blog
* branch master -> FETCH_HEAD
* [new branch] master -> origin/master
fatal: refusing to merge unrelated histories



















git pull origin master --allow-unrelated-histories:

Este comando permite hacer el merge con el repositorio remoto ya que este hace parte de una historia de archivos distinta a la que se tiene en local. "fatal: refusing to merger histories unrelated"

git push origin master:

Este comando sirve para subir los archivos al repositorio remoto.




Nota de buena práctica:

Antes de hacer un push siempre hacer un pull de origin (remoto) a master (local); antes de hacer el pull, hacer commit para que no se pierdan los cambios.




git checkout + nombre de la rama/de la versión:

Nos cambia el repositorio al de la rama o versión que le indiquemos.




git branch + nombre de la rama:

Este comando crea una rama del repositorio con el nombre que le indiquemos.




git push origin "rama":

Este comando envía los datos de la rama al repositorio remoto.
